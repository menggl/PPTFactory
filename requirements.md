重要：标准化生产ppt逻辑流程
```pgsql
master_template.pptx
   ↓（自动提取）
template_metadata.json
   ↓（加上 LLM）
LLM 自动选择 template_id
   ↓
复制模板页
   ↓
填充内容
   ↓
渲染风格策略
   ↓
导出最终 PPT
```


实现方案：模板引擎保持一个（统一），模板文件（不同风格）使用多套，渲染逻辑通过策略类/样式类进行扩展
- Template Engine（模板引擎） → 统一
- Templates（PPT 模板文件） → 多套
- LayoutStyle / StyleStrategy（风格策略类） → 多个

模板文件（Template） = 静态的布局结构（骨架）
例如：
- 文本框位置与大小
- 图片占位符位置、大小
- 字体大小、默认字号（基础值）
- 占位符位置、对齐方式
- 页面布局结构
模板本质上是一种 文档骨架。

策略类（Style Strategy） = 动态的风格规则（填充到骨架里面的内容的风格设定）
例如：
- 字体（宋体、黑体、无衬线等）
- 标题颜色 / 正文颜色
- 背景颜色 / 背景图
- 字体大小的微调、字体颜色、字体类型
- 行间距、段落间距
- 图标风格
- 页面配色主题
策略定义的是 风格、规则、渲染逻辑，不是页面结构。

将所有的模板都保存在templates/master_template.pptx这一个文件中，不管是语文、英语、数学、安全生产，都使用相同的这一个模板，只不过不同科目类别的风格策略不相同，对应的风格文件也不相同

风格（Style）文件包含下面的内容
1. 字体方案
标题字体（如 Source Han Sans）
正文字体（如 HarmonyOS Sans）
行距、段距、字距
2. 颜色方案（Color Palette）
主色（Primary）
副色（Secondary）
强调色（Accent）
背景色（Background）
3. 背景样式
是否用渐变？
是否用纯色？
是否用图片？
4. 元素外观
图片边框形状（圆角？直角？）
文本框圆角
图表配色风格
图标风格（线性图标、填充图标）
5. 动画风格（可选）
